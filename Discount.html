Discount System

A discount shop system created during my free time. This allowed me to practice Object Oreinted Programming during the summer holidays 2019.

About Discount System

The discount system was part of a challenege to create a system that uses Object Oreinted Programming. I used this opportunity to allow me to have a better understanding of object oriented programming, while also trying creative solutions to meet the requirements.

The Task

The task set was to create a shop that would calculate the items price after discont has been applied and the delivery cost. The shop will have a total of 5 items, with each item having its own discount type. These discount types are:

• None
• 2 for £20
• 3 for £10
• Buy 1 get 1 free
• 3 for the price of 2

The delivery charge is £7, however if all the items cost after discount is £50 or over, the delivery is free.

You can view the full task information here:
https://github.com/Yar890/Discount/blob/master/README.md

My Solution / Key Features

The key features of my solution to the problem is the following:

• Display all shop items and its discount type and normal price
• Allow the user to select what item they want to add to the basket
• Display all the items that are currently in the basket and its quantity
  (Example: "A: 2", "C: 4")
• Input validation (Example: User is allowed to enter "A", but not "&")
• Display delivery cost and total cost after discount has been applied

The Classes

The 3 classes that I decided to add are:

• Item: This class is used to create item objects, which contains information such as item name, price and discount type.

• Discount: This class is used to create new discount types. Each type of discount has its own class which inherits from the main "Discount" class. Each discount class has a applyDiscount method, which returns the cost of items after discount has been applied.

• Basket: This class is used to store what items are in the basket. This includes the AddToBasket, GetBasket and CalculateTotalPrice method.

Specific Improvements

Basket Improvements 
When looking at the requirements, I noticed that there could be a much better way of storing what items are in the basket. One posible solution is to store all the items that are in the basket, such as "AAABBCDD". However, the issue with this solution is that you would later need to work out the quantity of each item. I instead decided to use a dictionary and store the item and the quantity. For example, when I add a new item to the basket, I would only need to increase the quanityt of an item by 1. The dictionary looks similar to this: "A:3, B:2, C:1, D:2".

Discount Improvements
When looking at the different types of discounts, I noticed that a lot of the discounts offers are very similar, with the only difference being that each discount uses different numbers. For example, "2 for £20" offer is very similar to the "3 for £10" offer. This means that if both of these offers are very similar, I can use the same method for both these discounts, but use 2 different parameters for the offer quantity and offer price. In my discount system, I created 2 different discount classes, which are the following:

• "QuantityForSetPrice": This deals with offers such as "2 for £20" and "3 for £10".

• "BuyQuantityForPriceOfQuantity": This deals with offers such as "Buy 1 get 1 free" and "3 for the price of 2". The "Buy 1 get 1 free" offer is also the same as "Buy 2 for the price of 1".

This solution is much more useful when compared to other possible soutions, such as each item having its own discount calculations. My solution is very reusable, as a store with thousands of items can use the same discount type, but with different parameters.

2D Graphics

A HTML Canvas that is used to show 2D graphics. This was part of my 2D Graphics module during the first semester of second year University (Late 2019)

About 2D Graphics

The task set was to create a canvas with "whatever you want", as long as it meets the module specification. The specifications include:

• Creating Vector and Matrix objects, which both need to pass a set of unit tests provided to me

• Implement the Vector and Matrix objects to create translation, rotation and scale to objects on the canvas

• Implement a scenegraph system (This is a tree-like structure to organise what is shown on the scene)

• Implement a visitor pattern for the scenegraph system

• Include CPU independent animations

• Implement a variety of different collision detections (Such as circle and circle and circle to lines)

• Implement Physics and collision response

My 2D Canvas

For my 2D Graphics module, I decided that I would recreate a volleyball scene on a beach. I first created the Vector and Matrix objects aimed to pass all the unit tests, which I successfully managed to complete. 

After that, I started to work on scene graphs, including implementing transformation, rotation and scale, while using a visitor pattern. The objects that I created on the scene are the background image of a beach, 3 balls, a scaled-down sun and a volleyball net. All of the circular objects in the scene are made through a "makeCircle" method, which takes in a radius and returns a list of vectors, which can then be used to draw a circle. 

I then created a few different animations. The first animation made the sun rotate, while the other 3 animations make the balls in the scene move either right, left, up or down. To make the animations CPU independent, I kept track of the time between each frame that is drawn on the scene. This means that all changes made to the object will always run at the same rate, even if your system is running slowly.

The final element that I added to the canvas is collision detection between 2 different balls and the wall. When a collision is detected with the wall, the ball will then move in the opposite direction. When a collision between 2 balls is detected, both ball will stop moving.

Unfortunately, I was not able to implement the last specification of physics and collision response. However, with further free time, I would be able to add basic physics to the ball such as gravity and allow the ball to be thrown around the scene. If the ball were to collide with the net, I would make the ball go back in the opposite direction.
